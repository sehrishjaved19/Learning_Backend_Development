## ðŸ’» Client-Side JavaScript
### Purpose:
- Enables **interactive, dynamic user experiences** within the browser.

### Steps:
1. **UI Rendering:** The user interface is built using **HTML** and **CSS**.  
2. **User Action:** When the user selects an option, **JavaScript business logic** is triggered.  
3. **Request Sending:** The JavaScript application sends a **web service request** in **JSON format** over **HTTP**.  
4. **Server Handling:** A **RESTful web service** on the server intercepts the request.  
5. **Server Processing:** The request is processed by a **server-side application** (e.g., Java, PHP, etc.), which sends back a response to the client.

---

## ðŸ–¥ï¸ Server-Side JavaScript (Node.js)
### Purpose:
- **Node.js** allows JavaScript to run on the **server** instead of just in the browser.

### Process:
1. The user interacts with a UI built using **HTML** and **CSS**.  
2. The interaction triggers **client-side JavaScript** that handles business logic.  
3. The client application sends a **JSON-formatted web service call** over **HTTP**.  
4. A **REST web service** intercepts the HTTP request.  
5. Instead of Java or PHP, a **Node.js application** written in JavaScript handles the request on the **server**.

### Key Point:
- The **same language (JavaScript)** is now used both on the **client** and **server**, enabling **end-to-end development** in a single language.
